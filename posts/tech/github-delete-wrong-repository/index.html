删除本地仓库/错误的远程仓库关联

项目场景：#

Git bash

问题描述1：#

在错误的目录使用git init初始化了一个仓库，此时应该将该仓库删除

解决方案1：#

  1. 使用ls -a 命令显示隐藏文件夹和文件，检查该目录下是否有.git文件夹

  2. 发现该目录下含有.git文件夹，使用rm -rf .git删除.git文件夹，同时本地仓库也会被销毁

问题描述2：#

将本地仓库关联到错误的远程仓库，此时应该将该关联删除

解决方案2：#

  1. 查看本地仓库关联的远程仓库名称及URL
git remote -v
  1. 确认关联的错误远程仓库之后，删除关联
git remote remove <远程仓库名称>
